# Gemini API 호출 패턴 분석 및 429 오류 원인 분석

## 📊 현재 API 호출 패턴

### 1. 뉴스 번역 (translate_headlines)
- **호출 위치**: `src/crawler.py` → `translate_headlines()`
- **호출 방식**: `ai_researcher._call_ai(batch_prompt, max_retries=2)`
- **프롬프트 길이**: 짧음 (영어 뉴스 제목 + 요약만)
- **예상 토큰**: ~200-500 토큰
- **호출 빈도**: 1회 (배치 번역)
- **상태**: ✅ 정상 작동

### 2. 리포트 생성 (generate_briefing)
- **호출 위치**: `src/ai_researcher.py` → `generate_briefing()`
- **호출 방식**: `self._call_ai(prompt, max_retries=5)`
- **프롬프트 길이**: 매우 김 (전체 데이터 + 긴 지시사항)
- **예상 토큰**: ~5,000-15,000 토큰
- **호출 빈도**: 1회
- **상태**: ⚠️ 429 오류 발생 가능

## 🔍 429 오류 발생 원인 분석

### 원인 1: 프롬프트 길이 차이
```
뉴스 번역:   ~500 토큰  (짧음)
리포트 생성: ~10,000 토큰 (매우 김) ← 20배 차이
```

**영향:**
- 긴 프롬프트는 더 많은 토큰을 소모
- 무료 티어 할당량(일일 20회)을 빠르게 소진
- 토큰 기반 할당량도 고려해야 함

### 원인 2: 프롬프트에 웹 검색 요청 포함 (수정 완료)
**이전 프롬프트:**
```
"WEB을 통해 가장 최신의 실시간 정보를 검색"
"반복해서 10회 검증"
"실시간 WEB 검색 광범위하게 수행"
```

**문제점:**
- AI가 실제로 웹 검색을 시도할 수 있음
- 웹 검색은 추가 API 호출이나 토큰 소모 발생 가능
- "10회 검증"은 반복 작업을 유도할 수 있음

**수정 후:**
```
"⚠️ 중요: 웹 검색 기능을 사용하지 마십시오."
"제공된 입력 데이터만을 기반으로 분석하십시오."
```

### 원인 3: API 호출 간 지연 부족
**이전:**
- 번역 완료 → 즉시 리포트 생성
- 연속 호출로 Rate Limit 위반 가능

**수정 후:**
- 번역 완료 → 3초 대기 → 리포트 생성
- API 호출 전 2초 지연 (재시도 시)

### 원인 4: 할당량 관리 부족
**무료 티어 제한:**
- 일일 20회 요청 제한
- 번역 1회 + 리포트 1회 = 2회 사용
- 하지만 토큰 사용량이 많으면 할당량 초과 가능

## ✅ 적용된 개선 사항

### 1. FRED API 메시지 개선
```python
# 이전
return "[MACRO DATA (Source: Federal Reserve FRED)]\nFRED API 라이브러리 미설치"

# 개선 후
return "[MACRO DATA (Source: Federal Reserve FRED)]\n⚠️ FRED API 라이브러리(fredapi) 미설치 - yfinance 기반 지표로 대체됨"
```

### 2. 웹 검색 요청 제거
- 프롬프트에서 모든 웹 검색 관련 지시사항 제거
- "제공된 데이터만 사용" 명시

### 3. API 호출 간 지연 추가
- 번역과 리포트 생성 사이 3초 대기
- 재시도 시 2초 대기

### 4. 토큰 사용량 로깅
- 프롬프트 길이 및 예상 토큰 수 로깅
- 실제 토큰 사용량 추적

### 5. 프롬프트 최적화
- 불필요한 반복 요청 제거 ("10회 검증" 등)
- 명확한 지시사항으로 토큰 사용량 최소화

## 📈 예상 효과

### 할당량 사용 감소
- 이전: 웹 검색 시도로 인한 추가 토큰 소모
- 개선 후: 제공된 데이터만 사용하여 토큰 사용량 감소

### 429 오류 감소
- API 호출 간 지연으로 Rate Limit 위반 방지
- 프롬프트 최적화로 토큰 사용량 감소

## 🎯 추가 권장 사항

### 1. 할당량 모니터링
- 일일 API 호출 횟수 추적
- 토큰 사용량 모니터링
- 할당량 초과 전 경고

### 2. 프롬프트 최적화
- 불필요한 지시사항 제거
- 핵심 내용만 포함
- 토큰 사용량 최소화

### 3. 에러 처리 강화
- 429 오류 시 자동 재시도 (이미 구현됨)
- 할당량 초과 시 명확한 메시지 표시

## 📝 결론

**429 오류가 리포트 생성 시에만 발생하는 이유:**
1. ✅ 프롬프트가 매우 길어서 토큰 사용량이 많음
2. ✅ 웹 검색 요청이 포함되어 추가 토큰 소모 가능 (수정 완료)
3. ✅ API 호출 간 지연 부족 (수정 완료)
4. ✅ 할당량 관리 부족 (개선 완료)

**개선 효과:**
- 웹 검색 요청 제거로 토큰 사용량 감소
- API 호출 간 지연으로 Rate Limit 위반 방지
- 프롬프트 최적화로 효율성 향상

**참고:**
- 뉴스 번역은 짧은 프롬프트로 정상 작동
- 리포트 생성은 긴 프롬프트로 할당량 소진 가능성 높음
- 개선 사항 적용 후 429 오류 발생 빈도 감소 예상
