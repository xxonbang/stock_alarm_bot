# 429 에러 원인 검증 가이드

## 개요

429 에러의 원인을 찾기 위해 두 가지 기능을 순차적으로 비활성화하여 검증합니다:
1. 뉴스 번역 기능
2. 웹 검색 기능 강화 요청

## 환경변수 제어

### 뉴스 번역 기능 제어
- **환경변수**: `ENABLE_NEWS_TRANSLATION`
- **기본값**: `true` (활성화)
- **비활성화**: `ENABLE_NEWS_TRANSLATION=false`

### 웹 검색 기능 제어
- **환경변수**: `ENABLE_WEB_SEARCH`
- **기본값**: `true` (활성화)
- **비활성화**: `ENABLE_WEB_SEARCH=false`

## 검증 방법

### 방법 1: 자동 검증 스크립트 사용

```bash
python3 test_429_error_verification.py
```

이 스크립트는 두 가지 테스트를 순차적으로 실행합니다:
1. 뉴스 번역 비활성화 테스트
2. 웹 검색 비활성화 테스트

### 방법 2: 수동 검증

#### 테스트 1: 뉴스 번역 비활성화

```bash
# 뉴스 번역만 비활성화
export ENABLE_NEWS_TRANSLATION=false
export ENABLE_WEB_SEARCH=true

# 전체 워크플로우 실행
python3 src/main.py
```

**확인 사항:**
- 뉴스 번역이 실행되지 않는지 확인
- 리포트 생성 시 429 에러가 발생하는지 확인
- 로그에 "[검증 모드] 뉴스 번역 기능이 비활성화되어 있습니다" 메시지 확인

#### 테스트 2: 웹 검색 비활성화

```bash
# 웹 검색만 비활성화
export ENABLE_NEWS_TRANSLATION=true
export ENABLE_WEB_SEARCH=false

# 전체 워크플로우 실행
python3 src/main.py
```

**확인 사항:**
- 뉴스 번역은 정상 작동하는지 확인
- 리포트 생성 시 429 에러가 발생하는지 확인
- 로그에 "[검증 모드] 웹 검색 기능이 비활성화되어 있습니다" 메시지 확인

## 원상복구 방법

검증 완료 후 원래 상태로 복구:

```bash
# 환경변수 제거 (기본값 true로 복귀)
unset ENABLE_NEWS_TRANSLATION
unset ENABLE_WEB_SEARCH

# 또는 명시적으로 활성화
export ENABLE_NEWS_TRANSLATION=true
export ENABLE_WEB_SEARCH=true
```

## 예상 결과

### 시나리오 1: 뉴스 번역이 원인인 경우

- 테스트 2 (웹 검색 비활성화): ❌ 429 에러 발생

### 시나리오 2: 웹 검색이 원인인 경우

### 시나리오 4: 둘 다 원인이 아닌 경우
- 테스트 1 (뉴스 번역 비활성화): ❌ 429 에러 발생
