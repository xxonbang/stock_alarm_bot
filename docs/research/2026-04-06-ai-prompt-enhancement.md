> 목적: Gemini API 기반 주식 투자 리포트(경보/주간) 프롬프트 개선안 조사

## 1. 프롬프트 엔지니어링 기법

### 1-1. Chain of Thought (CoT) - 단계별 추론

**핵심 개념:**
복잡한 분석을 중간 추론 단계로 분해하여 LLM이 전문가 분석가처럼 논리적으로 사고하도록 유도하는 기법. 학술 연구에서 가장 많이 인용되는 프롬프팅 기법(84개 금융 연구 중 28건 참조).

**적용 방법:**
```
다음 단계를 순서대로 따라 SK하이닉스를 분석하세요:

1단계 - 데이터 확인: 제공된 주가/수급/매크로 데이터에서 핵심 변화를 식별
2단계 - 원인 분석: 각 변화의 원인을 추론 (실적? 업황? 매크로?)
3단계 - 영향 평가: SK하이닉스 실적/주가에 미치는 영향을 정량적으로 평가
4단계 - 시간축 구분: 단기(1주)/중기(1개월)/장기(3개월) 영향 분리
5단계 - 결론 도출: 종합 판단과 근거 요약
```

**FinCoT 기법 적용 (2025년 최신 연구):**
FinCoT는 전문가 추론 블루프린트를 Mermaid 다이어그램으로 프롬프트에 삽입하여 LLM이 도메인 전문가의 사고 흐름을 따르도록 하는 구조화된 CoT. 일반 모델의 정확도를 63.2% → 80.5%로 향상시키면서 출력 길이를 최대 8.9배 줄임.

```
<hint>
graph TD
  A[데이터 수신] --> B{이상 징후 있는가?}
  B -->|Yes| C[원인 분류: 실적/업황/매크로/수급]
  B -->|No| D[추세 지속 확인]
  C --> E[영향도 정량화: 매출/이익 영향 %]
  E --> F[시간축별 시나리오]
  F --> G[액션 권고]
</hint>
```

**실효성:** 높음 - 학술적으로 가장 검증된 기법. 금융 분석의 논리적 일관성 대폭 향상.
**구현 난이도:** 낮음 - 기존 프롬프트에 단계 구조만 추가하면 됨.

### 1-2. Tree of Thought (ToT) - 분기형 추론

**핵심 개념:**
CoT가 선형 추론이라면 ToT는 여러 추론 경로를 동시에 탐색하고 가장 유망한 경로를 선택. 의사결정 트리처럼 분기점에서 여러 가능성을 평가.

**적용 방법:**
```
SK하이닉스 투자 판단을 위해 3가지 독립적인 추론 경로를 탐색하세요:

경로 A - 수요 측면: HBM/AI 반도체 수요 → 삼성/SK 점유율 → 가격 전망 → 실적 영향
경로 B - 공급 측면: CAPEX 동향 → 공급 과잉 리스크 → 재고 수준 → 마진 영향
경로 C - 매크로 측면: 금리/환율 → 글로벌 IT 투자 → 반도체 사이클 위치

각 경로의 결론을 비교하고, 불일치하는 부분을 식별한 뒤 종합 판단을 내리세요.
```

**실효성:** 중간~높음 - 복잡한 투자 판단에 효과적이나 API 비용 증가.
**구현 난이도:** 중간 - 단일 프롬프트로 구현 가능하나 출력 길이 증가.

### 1-3. Self-Consistency - 다중 추론 합의

**핵심 개념:**
동일한 질문에 대해 여러 번 추론(temperature 높여서)한 뒤 다수결로 최종 답을 선택. 개별 CoT의 변동성을 줄이고 신뢰성 향상.

**적용 방법:**
```python
# 구현 예시 (코드 레벨)
responses = []
for i in range(5):
    response = gemini.generate(prompt, temperature=0.7)
    responses.append(extract_signal(response))  # "매수"/"관망"/"매도"

final_signal = majority_vote(responses)
confidence = agreement_ratio(responses)  # 5/5 합의 = 100%
```

프롬프트에서 직접 유도할 수도 있음:
```
이 분석을 3가지 다른 관점에서 독립적으로 수행하고,
각 관점의 결론이 일치하는지 확인하세요.
일치도가 낮으면 "의견 분산이 크다"고 명시하세요.
```

**실효성:** 높음 - 금융 분석에서 정확도와 해석 가능성 모두 향상. 규제 도메인에 적합.
**구현 난이도:** 중간 - API 다중 호출 필요 (비용 x N배). 단일 프롬프트 내 유사 구현도 가능.

## 2. 행동재무학 기반 편향 교정

**핵심 개념:**
AI도 학습 데이터의 편향을 반영하여 확증편향(기존 관점 강화), 최근성편향(최근 데이터 과대평가), 앵커링(첫 번째 수치에 고착)에 빠질 수 있음. 연구에 따르면 AI 가이드라인이 과신편향을 41.3%, 처분효과를 37.8% 감소시킴.

**적용 방법 - 편향별 교정 프롬프트:**

### 2-1. 확증편향 교정
```
분석 시 다음 규칙을 따르세요:
- 긍정적 데이터를 먼저 나열한 뒤, 반드시 동일한 분량의 부정적 데이터를 나열
- "하지만", "반면" 등으로 시작하는 반론을 최소 3개 포함
- 현재 포지션(보유 중)과 무관하게 중립적으로 분석
```

### 2-2. 최근성편향 교정
```
최근 1주일 데이터뿐 아니라 반드시 다음을 함께 고려하세요:
- 3개월/6개월/1년 추세와 비교
- "이번이 특수한 상황인지, 역사적으로 반복된 패턴인지" 명시
- 과거 유사 상황에서의 결과를 1개 이상 인용
```

### 2-3. 앵커링 교정
```
분석 시 특정 가격(목표가, 매수가)에 고정되지 마세요:
- 현재 가격 기준으로만 판단 (매수가 대비 수익률은 참고용)
- "현재 이 가격에서 새로 매수할 것인가?"라는 질문에 답하세요
- 목표가는 근거 없이 제시하지 말 것
```

### 2-4. 종합 편향 검사 프롬프트
```
분석을 완료한 후, 다음 편향 체크리스트를 자체 적용하세요:
□ 확증편향: 내 결론에 반하는 핵심 데이터를 무시하지 않았는가?
□ 최근성편향: 최근 1주 데이터에 과도하게 가중치를 부여하지 않았는가?
□ 앵커링: 특정 가격/수치에 결론이 고착되지 않았는가?
□ 생존자편향: 실패 사례를 고려했는가?
위반 항목이 있으면 해당 부분을 수정하세요.
```

**실효성:** 높음 - 연구 결과 정량적으로 편향 감소 확인됨. 투자 분석의 핵심 약점을 직접 공략.
**구현 난이도:** 낮음 - 시스템 프롬프트에 규칙 추가만으로 구현.

## 3. 투자 전문가 페르소나

**핵심 개념:**
LLM에 특정 투자 전문가의 역할을 부여하면 분석 스타일과 의사결정 패턴이 실제로 달라짐. GuruAgents 연구(2025)에서 동일 LLM(GPT-4o)에 다른 페르소나를 부여한 결과: 워런 버핏 에이전트 42.2% CAGR, 피오트로스키 30.9%, 벤저민 그레이엄 28.7% (벤치마크 S&P 500 26.3%).

**적용 방법:**

### 단일 페르소나 (경보 리포트에 적합)
```
당신은 반도체 섹터 전문 애널리스트입니다. 10년 이상 SK하이닉스를 커버한 경험이 있으며,
HBM/DRAM/NAND 시장의 공급-수요 사이클을 깊이 이해하고 있습니다.
기술적 분석보다는 업황/실적 기반 판단을 우선하되, 수급 데이터도 참고합니다.
```

### 다중 페르소나 (주간 심층 리포트에 적합)
```
다음 3가지 관점에서 순서대로 분석하세요:

[가치투자자 관점] PER/PBR/ROE 기준으로 현재 밸류에이션 적정성 판단
[모멘텀 트레이더 관점] 기술 지표와 수급 흐름 기반 단기 방향성 판단
[매크로 전략가 관점] 금리/환율/반도체 사이클 위치 기반 중장기 판단

마지막으로 3가지 관점의 합의점과 불일치점을 정리하세요.
```

**주의:** 수치적 추론이 요구되는 퀀트 분석에서는 페르소나 효과가 제한적. LLM의 수치 추론 능력 한계 때문.

**실효성:** 중간~높음 - 분석의 다양성과 깊이 향상. 다만 페르소나가 너무 강하면 편향 유발 가능.
**구현 난이도:** 낮음 - 시스템 프롬프트 변경만으로 즉시 적용.

## 4. 상관관계/인과관계 분석 유도

**핵심 개념:**
LLM은 기본적으로 level-1 인과 추론(직접 원인-결과)만 수행하며, 깊은 인과 체인(2차, 3차 효과)은 명시적으로 유도해야 함. 4차 이상 효과에서는 논리적 일관성이 급격히 떨어짐(GPT 기반 연구).

**적용 방법:**
```
다음 이벤트의 인과 체인을 3단계까지 분석하세요:

이벤트: [미국 AI CAPEX 확대 발표]

1차 효과 (직접): → SK하이닉스 HBM 수주 증가 예상
2차 효과 (간접): → DRAM 가격 상승 압력 → 경쟁사 CAPEX 확대 → 공급 확대 리스크
3차 효과 (파급): → 반도체 장비 수요 증가 → 소재 가격 상승 → 원가 부담

각 단계에서:
- 발생 확률 (높음/중간/낮음)
- 시간 지연 (즉시/1-3개월/3-6개월)
- SK하이닉스 영향 방향 (+/-)과 크기 (대/중/소)
를 명시하세요.
```

### "So What" 프롬프트 체인
```
모든 데이터 포인트에 대해 "그래서 SK하이닉스에 어떤 의미인가?" 질문을 적용하세요.
단순 사실 나열이 아닌, 투자 판단에 연결되는 해석을 제공하세요.

예시:
BAD: "미국 CPI가 3.2%로 예상보다 높았다"
GOOD: "미국 CPI 3.2%는 금리 인하 지연을 의미 → 반도체 기업 밸류에이션 부담 →
       다만 SK하이닉스는 AI 수혜 프리미엄으로 상대적 방어 가능 (HBM 매출 비중 40%+)"
```

**실효성:** 높음 - 리포트의 분석 깊이를 결정하는 핵심 기법. 3차 효과까지가 실용적 한계.
**구현 난이도:** 낮음 - 프롬프트 구조화만으로 구현.

## 5. 리스크 정량화 유도

**핵심 개념:**
"주의하세요"가 아닌 확률/영향도 매트릭스로 리스크를 구조화. 각 리스크에 발생 확률(1-5)과 영향도(1-5)를 부여하고 리스크 스코어(= 확률 x 영향도)로 우선순위화.

**적용 방법:**
```
현재 SK하이닉스가 직면한 리스크를 다음 형식으로 정량화하세요:

| 리스크 | 발생확률 (1-5) | 영향도 (1-5) | 스코어 | 시간축 | 조기경보지표 |
|--------|---------------|-------------|--------|--------|-------------|
| 예시: HBM 경쟁 심화 | 4 | 3 | 12 | 3-6개월 | 삼성전자 HBM3E 양산 뉴스 |

규칙:
- 확률: 1=거의 없음(<10%), 2=낮음(10-30%), 3=중간(30-50%), 4=높음(50-70%), 5=매우 높음(>70%)
- 영향도: 1=미미(<2% 주가), 2=소폭(2-5%), 3=보통(5-10%), 4=대폭(10-20%), 5=심각(>20%)
- 최소 5개 리스크 식별
- 스코어 15 이상은 "핵심 리스크"로 별도 강조
```

### 리스크-기회 균형 프롬프트
```
리스크와 동일한 형식으로 기회 요인도 정량화하고,
순 리스크 스코어 = 리스크 총합 - 기회 총합 을 계산하세요.
순 리스크가 양수면 방어적, 음수면 공격적 포지션을 권고하세요.
```

**실효성:** 높음 - 의사결정에 직접 활용 가능한 구조화된 출력. 보드 레벨 커뮤니케이션에도 유용.
**구현 난이도:** 낮음 - 출력 포맷 지정만으로 구현.

## 6. 멀티 에이전트 관점

**핵심 개념:**
TradingAgents 프레임워크(2024-2025)는 전문 분석가 팀 → 찬반 토론팀 → 트레이더 → 리스크 관리팀의 구조로 투자 의사결정을 시뮬레이션. 벤치마크 대비 누적수익률 23-27% vs 6% (3개월 테스트), 샤프비율 5.6-8.2 vs 2.3.

**프롬프트 레벨 구현 (멀티 에이전트를 단일 프롬프트로 시뮬레이션):**

```
다음 4명의 분석가가 순서대로 발언합니다. 각 분석가는 이전 발언을 참고하되,
자기 전문 영역의 관점을 우선합니다.

[기술적 분석가]: 차트 패턴, 기술 지표, 수급 흐름 기반 분석
[펀더멘털 분석가]: 실적, 밸류에이션, 경쟁력, 산업 사이클 기반 분석
[매크로 전략가]: 금리, 환율, 글로벌 경기, 지정학 리스크 기반 분석
[리스크 매니저]: 위 3명의 분석에서 간과된 리스크를 지적하고, 포지션 사이징 제안

마지막으로 [의장]이 합의/불일치 사항을 요약하고 최종 권고를 제시합니다.
```

### 찬반 토론 구조 (TradingAgents의 Researcher Team 모방)
```
위 분석 결과를 바탕으로 토론을 진행합니다:

[강세론자]: SK하이닉스 매수를 지지하는 가장 강력한 논거 3가지
[약세론자]: SK하이닉스 매수에 반대하는 가장 강력한 논거 3가지

[중재자]: 양측 논거의 강도를 평가하고 (각 1-10점), 종합 판단을 제시하세요.
```

**실효성:** 높음 - 단일 관점의 한계를 보완. 학술적으로 가장 높은 성과 개선 보고.
**구현 난이도:** 중간 - 단일 프롬프트로도 시뮬레이션 가능하나, 진정한 멀티 에이전트는 다중 API 호출 필요.

## 7. 뉴스 해석 깊이 - 2차/3차 효과

**핵심 개념:**
대부분의 AI 뉴스 분석은 1차 효과(직접 영향)에 머무름. 투자에서 알파는 2차/3차 효과(간접 영향)를 먼저 파악하는 데서 발생. 다만 LLM은 4차 이상 효과에서 논리적 일관성이 급격히 하락.

**적용 방법:**
```
다음 뉴스를 분석할 때, 반드시 2차/3차 효과까지 추론하세요:

[뉴스]: {뉴스 내용}

분석 형식:
■ 1차 효과 (직접, 즉시~1주): 이 뉴스가 SK하이닉스에 직접 미치는 영향
■ 2차 효과 (간접, 1주~1개월): 이 뉴스로 인해 변화하는 다른 요인이 SK하이닉스에 미치는 영향
■ 3차 효과 (파급, 1~3개월): 2차 효과의 연쇄 반응

각 효과에 대해:
- 영향 방향: 긍정(+) / 부정(-) / 중립(0)
- 확신도: 높음 / 중간 / 낮음
- 이미 주가에 반영되었는가: 예/아니오/부분적
```

### 뉴스 클러스터링 프롬프트
```
개별 뉴스가 아닌, 이번 주 뉴스들의 전체 내러티브를 파악하세요:
- 뉴스들이 공통적으로 가리키는 방향은 무엇인가?
- 서로 모순되는 뉴스가 있는가? (있다면 어느 쪽이 더 신뢰할 만한가?)
- 빠진 뉴스는 무엇인가? (보도되지 않은 것이 오히려 중요한 시그널일 수 있음)
```

**실효성:** 높음 - 리포트의 차별화 핵심. 다만 3차까지만 실용적.
**구현 난이도:** 낮음 - 프롬프트 구조화로 즉시 적용 가능.

## 8. 글로벌 매크로 내러티브

**핵심 개념:**
개별 지표(CPI, 금리, 환율) 나열이 아닌 "지금 시장의 큰 그림"을 먼저 그리고, 그 안에서 SK하이닉스의 위치를 정의. 글로벌 시장은 통계보다 내러티브에 의해 움직이는 경향이 강함.

**적용 방법:**
```
분석을 시작하기 전에, 먼저 현재 글로벌 시장의 "큰 그림"을 3문장으로 요약하세요:
1. 현재 시장을 지배하는 핵심 내러티브는 무엇인가? (예: "AI 골디락스", "스태그플레이션 공포" 등)
2. 이 내러티브가 지속될 것인가, 변화의 조짐이 있는가?
3. 이 큰 그림에서 반도체/SK하이닉스는 어디에 위치하는가? (수혜/피해/무관)

이후 분석은 이 큰 그림의 맥락 안에서 수행하세요.
개별 지표가 큰 그림을 강화하는지 약화하는지를 판단 기준으로 삼으세요.
```

### 시장 레짐 식별 프롬프트
```
현재 시장 레짐을 다음 중 하나로 분류하세요:
- Risk-On (위험 선호): 성장주/반도체 강세 예상
- Risk-Off (안전 선호): 방어주 강세, 반도체 약세 가능
- Rotation (순환): 섹터 간 자금 이동 중
- Confusion (혼조): 방향성 불명확, 관망 권고

분류 근거를 VIX, 금리, 달러인덱스, 크레딧 스프레드 중 2개 이상으로 뒷받침하세요.
```

**실효성:** 높음 - 개별 지표의 나열을 의미 있는 내러티브로 전환. 사용자 이해도 대폭 향상.
**구현 난이도:** 낮음 - 프롬프트 앞부분에 "큰 그림 먼저" 지시만 추가.

## 9. Confidence Calibration (확신도 보정)

**핵심 개념:**
LLM은 기본적으로 과신(overconfident) 경향이 있음. 자기 확신도를 명시적으로 표현하도록 강제하면 사용자가 AI 분석의 신뢰도를 판단 가능. 연구에 따르면 "설명을 먼저 생성한 뒤 답변"하면 보정 품질이 향상됨.

**적용 방법:**

### 확신도 강제 표현
```
모든 판단에 확신도를 0-100%로 표시하세요:
- 90%+: 높은 확신 (강한 데이터 근거, 역사적 패턴 일치)
- 70-89%: 중간 확신 (일부 불확실성 존재)
- 50-69%: 낮은 확신 (상충하는 데이터, 해석 여지 큼)
- 50% 미만: 매우 불확실 (추측에 가까움, 명시할 것)

규칙:
- 확신도 70% 미만인 판단은 반드시 불확실성의 원인을 명시
- "주가가 오를 것이다"와 같은 방향성 예측은 확신도 80% 이상일 때만 제시
- 확신도가 낮은 영역은 "모니터링 필요" 또는 "추가 데이터 필요"로 표시
```

### 과신 방지 프롬프트
```
다음 질문에 솔직하게 답하세요:
- 이 분석에서 가장 불확실한 부분은 무엇인가?
- 어떤 새로운 정보가 나오면 이 결론이 뒤집히는가?
- 이 판단의 근거가 되는 데이터의 품질/신뢰도는?
```

### 5th/95th Percentile 프레이밍
```
핵심 수치 예측 시 점 추정이 아닌 범위로 제시하세요:
- 낙관 시나리오 (상위 5%): [값]
- 기본 시나리오 (중앙값): [값]
- 비관 시나리오 (하위 5%): [값]
```

**실효성:** 높음 - 사용자의 의사결정 품질 직접 향상. 과신으로 인한 과도한 포지션 방지.
**구현 난이도:** 낮음 - 출력 형식 지정만으로 구현.

## 10. 기타 최신 연구/기법

### 10-1. ReAct 프레임워크 (Reasoning + Acting)

**핵심 개념:**
추론(Thought)과 행동(Action)을 교차 수행. TradingAgents에서 채택한 핵심 프레임워크로, 단순 추론이 아닌 "데이터 조회 → 추론 → 추가 조회 → 최종 판단" 사이클.

**적용 방법:**
```
다음 패턴으로 분석하세요:
Thought: 현재 데이터에서 가장 중요한 변화는 외국인 순매도 전환이다
Action: 외국인 순매도가 지속된 과거 사례의 이후 주가 흐름을 확인
Observation: 과거 5회 중 3회는 2주 내 반등, 2회는 추가 하락
Thought: 이번에는 반등 가능성이 높은데, HBM 수주 모멘텀이 건재하기 때문
Action: HBM 관련 최신 뉴스를 확인
...
Final: 종합 판단
```

**실효성:** 중간 - 구조적으로 깊은 분석 유도. 다만 현재 시스템이 실시간 데이터 조회를 지원해야 진정한 효과.
**구현 난이도:** 중간~높음 - 프롬프트만으로는 시뮬레이션, 실제 도구 연동 시 효과 극대화.

### 10-2. Analogy-Driven Financial CoT (AD-FCoT)

**핵심 개념:**
유사 상황 비유를 통해 추론을 유도. "2018년 메모리 다운사이클 때와 현재를 비교하라" 같은 역사적 유추.

**적용 방법:**
```
현재 상황과 가장 유사한 과거 사례 1-2개를 식별하고,
그때와 지금의 공통점/차이점을 분석한 뒤,
차이점이 결과에 어떤 영향을 미칠지 추론하세요.
```

**실효성:** 중간 - 역사적 맥락 제공에 유용. LLM의 학습 데이터에 해당 사례가 있어야 효과적.
**구현 난이도:** 낮음.

### 10-3. 구조화된 출력 강제 (Structured Output)

**핵심 개념:**
자유 텍스트가 아닌 JSON/테이블 형식으로 출력을 강제하면 분석의 일관성과 비교 가능성이 향상.

**적용 방법:**
```
분석 결과를 반드시 다음 JSON 형식으로 출력하세요:
{
  "market_regime": "risk-on | risk-off | rotation | confusion",
  "signal": "매수 | 관망 | 매도",
  "confidence": 0-100,
  "time_horizon": "1주 | 1개월 | 3개월",
  "key_risks": [{"risk": "설명", "probability": 1-5, "impact": 1-5}],
  "key_catalysts": [{"catalyst": "설명", "probability": 1-5, "impact": 1-5}],
  "narrative": "3문장 이내 요약"
}
```

**실효성:** 높음 - 자동화/후처리에 필수. 리포트 품질의 일관성 확보.
**구현 난이도:** 낮음.

### 10-4. Pre-mortem 분석

**핵심 개념:**
"이 투자가 실패했다고 가정하고, 왜 실패했는지를 거꾸로 추론하라." 미래의 실패 원인을 사전에 식별.

**적용 방법:**
```
6개월 후 SK하이닉스 주가가 20% 하락했다고 가정합니다.
가장 가능성 높은 원인 3가지를 추론하고, 현재 그 징후가 보이는지 평가하세요.
```

**실효성:** 중간~높음 - Devil's advocate와 유사하나, 더 구체적이고 시간축이 명확.
**구현 난이도:** 낮음.
