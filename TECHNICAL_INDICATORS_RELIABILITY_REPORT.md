## 🔍 데이터 소스 분석

### yfinance 라이브러리 특성

**장점:**
- 무료로 사용 가능
- 다양한 시장 데이터 제공
- 배당/분할 조정 지원 (`auto_adjust=True`)

**한계 및 주의사항:**
1. **데이터 지연**: 15-20분 지연 (실시간 데이터 아님)
2. **비공식 API**: 웹 스크래핑 기반으로 사이트 구조 변경 시 동작 중단 가능
3. **데이터 정확성**: 
   - 공식 거래소 데이터 대비 최대 11% 차이 가능
   - 일부 날짜에서 Open/High/Low/Close 값이 동일한 경우 발생 (비정상)
   - 주가 데이터 불일치 가능성
4. **Rate Limiting**: 과도한 요청 시 429 오류 또는 IP 차단 가능

**권장사항:**
- 비중요 분석에는 적합
- 실시간 거래나 고빈도 분석에는 부적합
- 정확성이 중요한 경우 공식 데이터 소스 사용 권장

## ✅ 기술적 지표별 검증 결과

### 1. RSI (Relative Strength Index)

**계산 방법**: Wilder's Smoothing (표준 방법)
- **위치**: `src/analysis.py` 라인 166-217
- **공식**: 
  - 초기값: 첫 14일의 단순 평균
  - 이후: `RMA = (이전 RMA × 13 + 현재 값) / 14`
  - `RSI = 100 - (100 / (1 + RS))`

**검증 결과**:
- ✅ 계산 로직 정확성: Wilder's 표준 방법 정확히 구현
- ✅ 수학적 정확성: 공식 검증 완료
- ⚠️ 데이터 소스 한계: yfinance 데이터 정확성에 의존

**실제 데이터 검증** (2026-01-14):
- 삼성전자 (005930.KS): RSI = 78.04
- KODEX 미국반도체 (390390.KS): RSI = 71.54

**주의사항**:
- yfinance 데이터의 정확성 한계로 인해 실제 시장 RSI와 최대 15포인트 차이 가능
- 전문 라이브러리(TA-Lib, pandas_ta) 사용 시 더 정확한 결과 가능

### 2. MA20 (20일 이동평균선)

**계산 방법**: 단순 이동평균 (Simple Moving Average)
- **위치**: `src/analysis.py` 라인 415
- **공식**: `MA20 = (최근 20일 종가 합계) / 20`

**검증 결과**:
- ✅ 계산 로직 정확성: pandas `rolling().mean()` 사용, 수학적으로 정확
- ✅ 실제 데이터 검증: 수동 계산과 함수 결과 일치 (차이 0.00원, 0.0000%)
- ✅ 데이터 일관성: `auto_adjust=True`로 배당/분할 조정 반영

**실제 데이터 검증** (2026-01-14):
- 삼성전자 (005930.KS): 
  - 현재가: 140,300원
  - MA20: 122,995원
  - 이격도: 114.07%
- KODEX 미국반도체 (390390.KS):
  - 현재가: 39,540원
  - MA20: 36,743원
  - 이격도: 107.61%

**웹 검색 검증**:
- 삼성전자 MA20 (2026-01-14): 웹 검색 결과 119,895원 (약 2.5% 차이)
  - **원인 분석**: yfinance 데이터 지연 또는 데이터 소스 차이 가능
  - **영향도**: 중간 (이격도 계산에는 큰 영향 없음)

## ⚠️ 발견된 문제점 및 개선 권장사항

### 1. yfinance 데이터 정확성 한계

**문제점**:
- 공식 거래소 데이터 대비 최대 11% 차이 가능
- 15-20분 데이터 지연
- 일부 날짜에서 비정상적인 데이터 패턴 (Open=High=Low=Close)

**영향도**: 
- **중간**: 상대적 지표(이격도, 52주 위치)에는 큰 영향 없음
- **높음**: 절대적 가격 분석에는 영향 있음

**개선 권장사항**:
1. 중요 분석에는 공식 데이터 소스 사용 고려
2. yfinance 데이터와 다른 소스(예: KRX API) 교차 검증
3. 데이터 정합성 체크 로직 추가 (예: High >= Low, Close가 High/Low 범위 내)

### 2. RSI 계산 방법

**현재 상태**: ✅ Wilder's Smoothing (표준 방법) 사용

**참고사항**:
- 일부 플랫폼은 EMA 방식 사용 (더 빠른 반응)
- Wilder's 방식은 더 안정적이지만 약간 느린 반응
- 현재 구현은 표준 방법이므로 적절함

### 3. auto_adjust 파라미터

**현재 상태**: ✅ `auto_adjust=True` 사용

**효과**:
- 배당/분할 조정 자동 반영
- 과거 가격과 현재 가격의 일관성 유지

**주의사항**:
- yfinance의 배당 조정이 일부 경우 부정확할 수 있음
- `repair=True` 파라미터 추가 고려 가능
